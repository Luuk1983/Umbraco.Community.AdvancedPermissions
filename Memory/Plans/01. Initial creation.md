I want to create an Umbraco (17) extension that give much more granular control over user role permissions.

The default Umbraco rols eystem is very limited:
- You can set the default permissions (like read, write, publish) on a user role
- On individual nodes, you can override these default permissions.
- When you apply multiple roles to a user, permissions are only added; there is no way to limit or retract a permissions, like a 'deny'.

The limitations are obvious:
- There is no inheritance
- It makes it really hard to do things like: I want a user to not being able to delete a news overview, but allow any children to be deleted.
- It's impossible to mark specific nodes as 'non deletable'. I really would like to protect vital nodes for getting deleted for instance.

So I would like a security system more like Sitecore:
- You can set permissions on a node. We can stick to the current permissions that Umbraco has, but we probably need an additional 'inherit' permission. You have three options per permission:
    - These permissions only apply to this specific node, like the current behaviour
    - These permissions apply to this node and any children
    - You can set seperate permissions for the node specifically and permissions for the children of the node.

You can set explicit allows and denies on these permissions. If not set, they become implicit allow and deny based on the inheritance.
The order which determines the final permissions is this order:
- An explicit deny tops verything
- Then an explicit allow
- Then an implicit deny
- Then an implicit allow

This makes sure it's safe by default.

I also want to add a 'everyone' role to Umbraco. This role is implcitly added and applies to every single user of Umbraco. This allows us to set explicit denies on important nodes. And since explicit denied top everything, they cannot be overriden.

I want to build this as an Umbraco plugin/package that can be installed. IT should use as much native functionality as possible. I want to minimize the risk that Umbraco updates break the package. So I also want to copy Umbraco functionality as little as possible. However, it will be a balance between functionality and how much we need to change. So I want to hear alternatives if they make sense, even if they would break Umbraco default functionality.

BEcause this is such an essential part of Umbraco, performance is essential.

We also need to come up with a way to manage these permissions. I'm looking at Sitecore again. If you have better suggestions, I would love to hear it, but Sitecore did it like this:
Sitecore had a 'security editor'. It shows the entire content tree (collapsed) and after each page, a list of permissions was shown. It showed the actually set permissions, so the implicit permissions would simply be neutral. You could click on the permission to edit it. Sitecore also has an access viewer that visually looks the same, but in this case, you can pick a user or role and see the actual permissions. When you click on a permission, you can see what role or roles influence that permission and why the effective permission is the way it is.

I feel like we need something like this approach to make it work. We also need default permissions for the 'root' node of content.

It's out of scope for this plan, but I probably want to use the same functionalty for media. Since media is in the end also just nodes, this should be possible.
